LOAD DATA INFILE REPLACE IGNORE

Query ini digunakan untuk menghandle duplikasi data pada kolom yang memiliki attribute primary key dari hasil load.

  • Jika kita menambahkan LOAD DATA INFILE dengan REPLACE maka data duplikat tersebut akan di timpah dengan nilai baru.
  • Namun jika kita menambahkan LOAD DATA INFILE dengan IGNORE maka nilai yang terdapat duplikat akan di skip.
  • Jika REPLACE atau IGNORE tidak ditulis ama sekali pada Query LOAD DATA INFILE maka mysql tidak akan mengeksekusi querynya atau error;

Berikut contoh penerapannnya:

  • Membuat table provinsi dengan kode_iso sebagai primary key:
    MariaDB [indonesia]> CREATE TABLE provinsi (
    -> nama_prov VARCHAR(50),
    -> kode_iso CHAR(2),
    -> ibukota VARCHAR (50),
    -> populasi INT,
    -> luas DEC(8,2),
    -> apbd DEC(8,2),
    -> PRIMARY KEY(kode_iso)
    -> );
